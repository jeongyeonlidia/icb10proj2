# 인력 턴오버(퇴사율) 분석 비즈니스 데이터 마트(Data Mart) 설계서

본 문서는 사람인 채용공고 크롤링 데이터를 기반으로 인력 턴오버(퇴사율)가 높은 기업 및 직무의 특징을 도출하기 위한 비즈니스 데이터 마트의 설계 요건, 피처 엔지니어링 로직, 그리고 산출 수식을 정의합니다.

---

## 1. 배경 및 목적

전통적인 채용 분석은 단순히 채용 건수나 지원자 수 같은 표면적인 볼륨에 집중하는 경향이 있습니다. 그러나 인사팀(HR) 관점에서 보다 중요한 문제는 **"채용된 인력이 얼마나 안정적으로 유지되는가"** 및 **"특정 부서에서 사람이 계속 이탈하여 메우기 식 채용을 반복하고 있는가(회전문 현상)"**입니다.

본 데이터 마트는 수집된 공고의 시계열 및 텍스트 특징을 피처 엔지니어링하여 다음 사항을 파악할 수 있도록 돕습니다:
1. 단순히 확장 성장을 하는 기업과 퇴사율이 높아 공고가 자주 바뀌는 블랙기업의 분류
2. 마감 직후 짧은 주기 안에 재등록이 반복되는 **'독박 직무(Toxic Rotation)'**의 식별
3. 턴오버가 빈번한 기업들이 올리는 **직무기술서(JD)의 텍스트 패턴 및 특징** 포착

---

## 2. 데이터 마트 스키마 및 피처 정의

본 데이터 마트는 원본 사람인 채용 데이터 600건과 **1:1 결합 관계**를 유지하며, M:N 병합 증폭 오류를 방지하기 위해 공고의 고유 URL인 `link`를 유일한 식별 키(Primary Key)로 설정하여 빌드되었습니다.

| 컬럼명 | 데이터 타입 | 설명 | 산출 수식 / 매핑 기준 |
| :--- | :---: | :--- | :--- |
| `company` | String | 기업명 | 원본 필드 |
| `company_type` | String | 기업 규모 및 유형 (대기업, 일반기업 등) | 원본 필드 |
| `employee_count` | Integer | 국민연금 가입 실제 종사자 수 | 국민연금공단 실데이터(2026-05-21 기준) 가입자 수 연동 매핑. 매칭 실패 시 기업 규모별 가상 폴백 수치[대기업(1500) 등] 적용 |
| `primary_sector` | String | 대표 직무 분류 | 공고 sectors 리스트 중 첫 번째 항목 파싱 |
| `title` | String | 채용 공고 제목 | 원본 필드 |
| `registration_date` | Date | 공고 최초 등록일 | 기준일(2026-06-20) - `reg_days_ago` (일전 등록 역산) |
| `end_date` | Date | 공고 마감일 | `registration_date` + `posting_period_days` (유지일수) |
| `posting_period_days` | Integer | 공고 게시 유지 기간 (일수) | 마감 기한(`D-X`, `~MM.DD`) 기반 역산 값 (기본 결측 시 12일로 폴백) |
| `reposting_interval_days` | Float | 이전 공고 마감 후 새 공고가 올라올 때까지의 간격 | 동일 기업-직무 내에서 $등록일_{t} - 마감일_{t-1}$ (일수) |
| `company_sector_postings` | Integer | 동일 기업 및 직무별 총 등록 공고 건수 | $Count(Company, Primary\_Sector)$ |
| `rotation_index` | Float | **공고 회전문 지수 (Rotation Index)** | $\frac{\text{company\_sector\_postings}}{\text{employee\_count}}$ |
| `is_toxic_rotation` | Integer | **독박 직무 판정 여부 (1 or 0)** | 동일 기업-직무 조합 내에서 `reposting_interval_days`가 14일 이하인 케이스가 1회 이상 존재할 경우 1, 없으면 0 |
| `toxic_jd_score` | Integer | JD 내 추상적/기피 키워드 출현 스코어 | '가족', '가족같은', '열정', '급구', '긴급', '인내', '보조', '초보', '단순', '야근' 포함 갯수 |
| `jd_copy_paste_ratio` | Float | 동일 회사 내 직전 공고와의 제목 유사도 | 단어 수준에서의 자카드 유사도(Jaccard Similarity) 계산 |
| `turnover_risk_score` | Float | **통합 턴오버 위험 점수 (100점 만점)** | (정규화된 `rotation_index` + `is_toxic_rotation` + `jd_copy_paste_ratio` + `toxic_jd_score`) 각 25% 가중 합산 |
| `turnover_risk_level` | Category | 위험 수준 등급 (Low, Medium, High) | `Low`(0~30점), `Medium`(30~60점), `High`(60~100점) 분류 |
| `link` | String | 채용 공고 고유 링크 (유일 키) | 원본 필드 |

## 3. 핵심 피처 엔진 설계 상세

### 3.1 회사별 공고 회전문 지수 (Rotation Index)
* **방향성**: 단순히 채용 규모가 커서 공고를 많이 내는 대기업(건전 성장형)과 규모 대비 비정상적으로 구인 공고를 빈번하게 갈아치우는 블랙기업을 식별하기 위함입니다.
* **비즈니스 가치**: 이 지수가 시장 평균이나 동종 규모의 타 기업군보다 높을수록, 직원들이 조직에 안착하지 못하고 자주 이탈(퇴사)하여 땜질식 채용을 수행하고 있음을 정량적으로 드러냅니다.

### 3.2 공고 재등록 주기 (Interval) 및 독박 직무 (Toxic Rotation)
* **방향성**: 시장 평균 공고 게시 기간의 중앙값(12일)을 기저값으로 상정하고, 동일 회사에서 직무별로 공고가 새로 고쳐 올라오는 주기(Time Delta)를 측정합니다.
* **비즈니스 가치**: 공고가 한 번 닫혔다가(채용 완료 또는 중단) 단 **2주 이내(14일 이하)**에 정확히 같은 직무의 공고가 다시 등록된다는 것은 **'채용 실패'** 또는 **'입사 후 초고속 조기 퇴사'**가 발생했음을 지시합니다. 이 현상이 9일간의 관찰 범위 내에서 단 1회 이상 확인되면 회전문 순환이 시작된 독박 직무 세그먼트로 판정합니다.

### 3.3 JD 텍스트 마이닝 & 복사-붙여넣기 지수
* **방향성**: 잦은 퇴사로 골머리를 썩는 기업들의 JD(직무기술서)가 가지는 언어적, 형태적 문제점을 잡아냅니다.
* **비즈니스 가치**:
  - **자카드 유사도**를 통해 산출한 `jd_copy_paste_ratio`가 1.0에 수렴한다는 것은 기업이 직무의 변경이나 구체적인 필요 분석 없이 과거의 공고를 그대로 복사하여 무작위 수시 채용을 남발하고 있음을 의미합니다.
  - 구체적인 요구 요건을 명시하기보다 '가족같은 분위기', '열정', '급구' 등의 추상적이거나 업무 기피를 유도하는 단어를 다수 남용하는 기업일수록 `toxic_jd_score`가 높게 산출됩니다.

  - **Medium (30~60점)**: 105건 (공고 마감 후 즉시 리프레시 등록된 독박 직무 군집)
