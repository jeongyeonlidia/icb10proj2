# 네이버 오픈API 공통 가이드

네이버 오픈API는 외부 개발자가 네이버 플랫폼의 기능(검색, 로그인, 데이터랩 등)을 직접 연동할 수 있도록 웹 인터페이스 또는 SDK 형태로 제공하는 서비스입니다. 본 가이드는 네이버 오픈API를 활용하여 애플리케이션을 개발할 때 필요한 인증 방식, 호출 규격, 공통 오류 처리에 관한 정보를 담고 있습니다.

---

## 1. 오픈API 종류 및 인증 방식

네이버 오픈API는 인증 여부 및 로그인 주체에 따라 크게 두 가지 유형으로 분류됩니다.

### A. 비로그인 방식 오픈 API (Client ID/Secret 기반)
* **특징**: 네이버 계정의 개별 사용자 로그인 없이, 애플리케이션 식별자만을 활용하여 호출하는 API입니다.
* **사용 API**: 검색(블로그, 뉴스, 카페글, 쇼핑 등), 데이터랩(통합 검색어 트렌드 등), Papago 번역 등
* **인증 방법**: HTTP 요청 헤더에 개발자 센터에서 발급받은 클라이언트 아이디와 시크릿을 실어 보냅니다.
  ```http
  X-Naver-Client-Id: {애플리케이션 등록 후 발급된 클라이언트 ID}
  X-Naver-Client-Secret: {애플리케이션 등록 후 발급된 클라이언트 Secret}
  ```

### B. 로그인 방식 오픈 API (OAuth 2.0 기반)
* **특징**: 개별 네이버 사용자의 동의 및 로그인이 필요한 기능에 접근하는 API입니다.
* **사용 API**: 네이버 로그인, 카페 글쓰기, 캘린더 등
* **인증 방법**: OAuth 2.0 인증 프로토콜을 통과하여 발급받은 `Access Token`을 헤더에 실어 보냅니다.
  ```http
  Authorization: Bearer {Access Token 값}
  ```

## 2. 개발 준비 프로세스

네이버 오픈API를 사용하려면 네이버 개발자 센터에서 사전 준비가 필요합니다.

1. **개발자 등록 및 로그인**: [네이버 개발자 센터](https://developers.naver.com/)에 접속하여 네이버 아이디로 로그인합니다.
2. **애플리케이션 등록**: `Application > 애플리케이션 등록` 메뉴로 이동합니다.
   * **애플리케이션 이름**: 서비스 구분을 위한 고유 명칭 입력 (최대 40자).
   * **사용 API**: 연동하고자 하는 API 서비스(예: 검색, 데이터랩 등)를 선택 및 추가합니다.
   * **서비스 환경**: 웹(URL), Android, iOS 등 실제 연동 환경을 등록합니다.
3. **인증키 확인**: 등록이 완료되면 `Application > 내 애플리케이션`의 개요 탭에서 **Client ID**와 **Client Secret**을 확인할 수 있습니다.

## 3. 공통 에러 메시지 형식

API 호출 시 오류가 발생하면 HTTP 상태 코드와 함께 JSON 혹은 XML 포맷의 에러 메시지가 반환됩니다.

### JSON 에러 형식 예시
```json
{
  "errorMessage": "Authentication failed. (인증에 실패했습니다.)",
  "errorCode": "024"
}
```

### 공통 오류 코드 목록

| 에러 코드 (errorCode) | HTTP 상태 코드 | 메시지 / 원인 | 조치 사항 |
| :--- | :--- | :--- | :--- |
| **024** | 401 | Authentication failed | Client ID 혹은 Client Secret 값이 일치하지 않는 경우 헤더 값을 확인합니다. |
| **028** | 401 | Authentication failed | API 권한이 없는 경우입니다. 개발자 센터 내 애플리케이션의 'API 설정'에서 사용하려는 API가 활성화되어 있는지 확인합니다. |
| **404** | 404 | Invalid API | 요청한 API URL 주소에 오타가 있는지 또는 존재하지 않는 엔드포인트인지 확인합니다. |
| **500** | 500 | System Error | 네이버 시스템 내부 오류입니다. 일시적인 장애일 수 있으므로 잠시 후 재시도하거나 개발자 포럼을 확인합니다. |
